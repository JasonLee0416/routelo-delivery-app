# RouteLO 개발 로드맵

이 문서는 현재 `main` 브랜치의 실제 구현을 기준으로 완료된 기능과 다음 작업을 정리한다.

## 제품 방향

- Android 우선 배달 업무 관리 앱
- Material Design 3 기반 전문 물류 대시보드 (라이트/다크 테마 전면 지원)
- 사용자 선택 내비 앱(티맵·카카오맵·네이버 지도) 길찾기 핸드오프
- 한국어 인수증 촬영 및 구조화 OCR (온디바이스 PP-OCRv5)
- 엄수 마감·예식 시간·지연 위험 우선 표시
- 로컬 우선 저장, 실사용 시작 시 빈 상태(샘플 데이터 없음)

## 현재 완료된 기능

### 운영 화면

- [x] 홈 대시보드
  - 오늘 전체·완료·남은 배달
  - 가장 가까운 엄수 마감 및 예식 시간
  - 업무 진행률과 남은 예상 거리
- [x] 배달 목록

- [x] 배달 동선 스택

  - 긴급 마감, 예식 시간, 지연 위험, 경로 변경
- [x] 설정 화면 (v2)
  - 그룹형 IA, 검색형 구역 수수료, 알림/경로/개인정보/내비 앱 설정
- [x] Material Design 3 하단 내비게이션
- [x] 라이트/다크 테마 전면 적용 (테마 컨텍스트 기반)

### 내비게이션 핸드오프

`app/services/ocr.ts`는 온디바이스 PP-OCRv5(ONNX) 런타임에 연결돼 있다. 앱 코드에는 더 이상 샘플/데모 데이터가 없으며, 데모 인수증 텍스트는 테스트 전용 픽스처로만 존재한다.

### 주소와 지도

## 공동 개발 규칙

1. 작업 전에 GitHub Issue를 작성하고 담당자를 지정한다.
2. 최신 `main`에서 기능 브랜치를 생성한다.
3. 한 PR은 한 기능 또는 한 문제만 다룬다.
4. 다른 미병합 브랜치에 의존하는 stacked PR은 최소화한다.
5. PR에 변경 이유, 테스트 방법, 화면 캡처, 충돌 가능 파일을 작성한다.
6. TypeScript와 Expo Doctor 통과 후 리뷰를 요청한다.
7. 승인된 PR만 `main`에 병합한다.
8. 병합 후 기능 브랜치를 삭제하고 관련 Issue를 닫는다.
